# External skills manifest — one skill per line, pinned to an exact version.
#
# Format:  <git-clone-url>@<tag-or-commit-sha>  [<path-in-repo>]  [<tree-sha>]
#
#   url@version    Pin to a tag or full commit SHA, never a branch, so installs
#                  are reproducible and updates are an explicit edit here.
#   path-in-repo   Only for skills that live in a subdirectory of a larger
#                  repo; fetched with a sparse checkout. The skill is named
#                  after the path's basename (otherwise the repo's).
#   tree-sha       Optional integrity pin: the git tree object of the skill
#                  directory — `git rev-parse '<version>:<path>'`, or
#                  '<version>^{tree}' for whole-repo skills. Tags can be
#                  re-pointed upstream; a tree sha cannot. fetch-skills.sh
#                  refuses to install content that doesn't match.
#
# Name/description are NOT recorded here — they live in the fetched SKILL.md.
#
# scripts/fetch-skills.sh clones each entry into skills/external/ (gitignored —
# external skills are referenced, never vendored/committed, and retain their
# own licenses).
#
# Example:
# https://github.com/example-org/example-skill@v1.2.0
